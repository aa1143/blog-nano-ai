# 第一性原理？其实你初中就学过了

马斯克说他造火箭用的是"第一性原理"。

于是这个词火了。

于是所有人开始觉得——这是天才才能用的思维方式，跟我没关系。

但我要告诉你：**你初中就已经掌握了它，只是不知道它叫这个名字。**

它有另一个名字，叫**因式分解**。

---

## 第一性原理是什么来头？

这个词不是马斯克发明的，最早来自亚里士多德：

> "在每一个系统的探索中，存在第一性原理——一个最基本的命题或假设，不能被省略或删除，也不能被违反。"

翻译成人话就是：**找到一件事最底层的、不可再拆的基本单元。**

听起来很玄。但你看这个：

```
12 = 2 × 2 × 3
```

这就是因式分解。把 12 拆到不能再拆——质因数。

**第一性原理，就是对现实问题做因式分解。**

---

## 用面包举个例子

面包怎么来的？

```
面包
└── 淀粉 × 水 × 酵母
      └── 淀粉 = 小麦加工
            └── 小麦 = 土壤 × 阳光 × 水分
                        └── 阳光 = 光子  ← 到底了，base case
```

你一直在问"为什么"，一直往下拆，拆到拆不动为止。

这个过程，在计算机里有个专有名词叫 **DFS，深度优先搜索**。

从根节点出发，不断往最深处走，直到触底，再把结果一层层 return 回来。

普通人遇到问题用的是 **BFS，广度优先**——把所有能想到的东西平铺一遍，停在第一层，看起来懂了很多，其实没有穿透任何一条链路。

**第一性原理，就是强制切换成 DFS。**

## 后台工程师每天都在做这件事

你可能觉得这还是太抽象。那我给你举个最接地气的例子。

后台开发工程师，每天接到的需求可能就一句话：

> **A 给 B 转 100 块。**

听起来两步：扣钱、加钱。

但真正开始做，就是一次因式分解的过程：

**第一层拆：钱怎么动？**
扣钱和加钱必须同时成功，不能一个成功一个失败。
→ 逼出了**事务**的概念。

**第二层拆：事务不够用了**
A 和 B 可能在不同的数据库、不同的服务，单机事务管不住。
→ 逼出了**分布式一致性**，CAP 理论，最终一致性。

**第三层拆：网络抖了怎么办？**
钱扣了，但对方加钱的请求没到。
→ 逼出了**幂等设计**，同一笔交易不能重复执行。

**第四层拆：账对不上怎么办？**
→ 逼出了**对账系统**，每天凌晨跑批把所有流水对一遍。

**第五层拆：双十一流量来了**
→ 逼出了**限流、降级、异步队列**。

```
A 给 B 转账
├── 原子性 → 事务
├── 跨服务 → 分布式一致性
├── 网络抖动 → 幂等
├── 数据核验 → 对账
└── 高并发 → 限流 × 降级 × 队列
```

每一层都是一次因式分解。每当你以为"这个做完了"，下面就又炸出一个新问题。

这和马斯克拆电池成本，**本质上是完全一样的思维动作**。

---

## 马斯克到底做了什么？

马斯克问：电池为什么那么贵？

所有人说：电池就是这个价，市场价嘛。

马斯克没接受这个答案，继续拆：

```
电池成本
├── 碳
├── 镍  
├── 铝
├── 钢
└── 加工费
```

原材料加在一起，比市场价便宜得多。

结论：自己买原材料组装。

这就是 Tesla 电池成本被打下来的逻辑。他没有什么魔法，就是**对"理所当然"的答案多问了几层为什么**。

---

## 因式分解有一个递归没有的含义

最后说一个细节，也是第一性原理最容易被忽视的地方。

因式分解不只是"拆"，更重要的是：

```
12 → 2 × 2 × 3
2 × 2 × 3 → 12  ✓
```

**拆完之后，乘回去结果不变。**

结构没有被破坏，只是被看穿了。

这就是第一性原理和"随便质疑一切"的本质区别——不是为了拆而拆，而是**拆完之后你能重新组装**，而且组装出来的东西比原来更清晰、更可控。

马斯克拆完电池成本，重新组装出了更便宜的电池。
工程师拆完转账需求，重新组装出了整个支付系统。

---

## 所以

第一性原理之所以被说得那么玄，本质上是两件事叠加：

1. **翻译腔的名字**太唬人，"第一性原理"听着像物理学术语
2. **马斯克的光环**让人觉得这是天才专属

但拆掉这两层包装，剩下的就是一句话：

> **遇到问题，不接受"就是这样"的答案，一直问为什么，直到问不下去。**

小孩子天生会这个，叫"为什么为什么为什么"。

工程师每天在做这个，叫"需求评审和技术方案"。

马斯克也在做这个，只是他在不同的问题上做，做出了不同的结果。

**这不是天才思维。这是初中数学。**

01 · 这个词本身就有问题

"第一性原理"——光听这个名字，就已经把 90% 的人劝退了。

翻译腔、学术感、高不可攀。但它的英文原文 first principles thinking，直译过来其实只是一件事：

把一件事拆到不能再拆，然后从底层重新搭起来。

就这一句话。不需要"第一性"三个字。

02 · 马斯克的光环陷阱

这个词之所以让人望而却步，很大程度上是因为马斯克。

一听马斯克，脑子里自动浮现——造火箭、造电动车、殖民火星。于是"第一性原理"在潜意识里就跟天才思维绑定了。

但马斯克自己举的例子呢？

电池为什么这么贵？大家都说就是这个价。那原材料到底值多少？碳、镍、铝……算下来便宜得多。那我们自己买原材料组装不就行了？

—— 马斯克谈 Tesla 电池成本

朴素得让人意外。他不是因为天才才用第一性原理，他是用了这个思路之后才做成了那些事。

大多数人把顺序搞反了：

× 误解的顺序
错误

天才 → 会用第一性原理 → 做成大事

✓ 真实的顺序
正确

遇到问题 → 不接受"就是这样"的答案 → 一直问为什么 → 做成大事 → 被称为天才

03 · 用编程思维理解它

如果你写过代码，这个概念其实一秒就懂：

第一性原理 = 递归 + DFS（深度优先搜索）

遇到一个问题，不停往下拆，拆到 base case（无法再拆的底层），然后把结果一层层 return 回来，重新搭建整个认知体系。

大多数人用的是 BFS（广度优先）——把所有能想到的东西平铺一遍，停在第一层，广而不深。看起来懂了很多，其实没有穿透任何一条链路。

04 · 后台开发每天都在做

还有个更接地气的例子：后台开发工程师。

做一个微信支付的转账功能，需求就一句话——A 给 B 转 100 块。

往下拆，每层都炸出新问题
L1
钱怎么动？

扣钱和加钱必须是原子操作 → 逼出了事务

L2
事务不够用了

A 和 B 在不同数据库 → 逼出了分布式一致性

L3
网络抖了怎么办？

钱扣了但对方没收到 → 逼出了幂等设计

L4
对完账发现不对？

→ 逼出了对账系统

L5
双十一流量来了

→ 逼出了限流、降级、异步队列

这和马斯克拆电池成本，本质上是完全一样的思维动作。只不过一个在造火箭，一个在写代码。

你每天都在用，
只是没人这么叫它
第一性原理不是天才的专属 · 它只是"继续问为什么"

# AI时代，不懂Markdown就像不会用标点符号

打开ChatGPT、Claude、Gemini、Kimi……

你有没有注意到，这些AI回答问题的时候，几乎都长一个样子——

重要的内容会**加粗**，回答会分成好几个板块，每个板块上面有个标题，细节用列表一条条列出来，代码单独放在灰色框里。

这不是AI的审美偏好，也不是产品设计的风格统一。

背后的原因很简单：这些AI在用**Markdown**回答你。

为什么？因为这些大模型的训练数据里，充斥着海量Markdown格式的内容——GitHub上数以亿计的项目文档、全球技术博客、Stack Overflow的问答帖子……AI就是在这种"语言环境"里长大的。Markdown是它最熟悉的表达方式，某种程度上，也是它的母语。

这就带出一个很多人没意识到的问题：

既然AI输出用的是Markdown，那你输入的提示词，是不是也该用Markdown来写？

答案是：**是的，而且差距大得出乎意料。**

同样的问题，散文式提问和Markdown结构化提问，AI给出的回答质量完全不在一个水平线上。前者靠AI猜你的意图，后者让AI精准执行你的要求。

这篇文章，带你从零了解Markdown：它从哪来、核心语法是什么、用什么工具、去哪里学。

10分钟读完，今天就能开始用。

## 它是怎么来的

时间拨回2004年。

那时候在网上写内容，要么用Word（格式绑死在软件里，换台电脑打开可能全乱），要么手写HTML标签（想加个标题要打 `<h1>标题</h1>`，想加粗要写 `<strong>文字</strong>`，繁琐到让人崩溃）。

程序员John Gruber受够了。他和黑客Aaron Swartz一起，设计了一套极简规则：**用普通符号代替复杂标签**，写的时候一眼就能读懂，发布的时候自动转成格式好看的网页。

他们把这套规则叫做Markdown——故意对应"markup"（标记语言）这个词，意思是：把这件事往简单了做，往下降，而不是往上堆。

一个名字，藏着整个设计哲学。

之后二十年，它悄悄渗透进整个互联网：

- **2008年**：GitHub选择它作为项目文档默认格式，全球开发者开始日常使用
- **2014年**：CommonMark规范出现，混乱的语法差异得到统一
- **2016年后**：Notion、飞书、语雀相继支持，普通人也开始接触
- **2022年至今**：ChatGPT横空出世，Markdown直接成了人和AI之间最高效的沟通桥梁

二十年前，它是程序员的小圈子工具。现在，它是AI时代每个人都该掌握的基本功。

## 核心语法，这些就够了

不需要全学。掌握下面这些，覆盖日常使用的80%场景。

**标题**

```
# 一级标题
## 二级标题
### 三级标题
```

`#` 后面记得加空格，几个 `#` 代表几级标题，最多用到三级就够了。

**加粗、斜体、删除线**

```
**这是加粗**
*这是斜体*
~~这是删除线~~
```

**列表**

```
无序列表：
- 第一条
- 第二条

有序列表：
1. 第一步
2. 第二步

任务清单：
- [x] 已完成
- [ ] 待完成
```

**引用块**

```
> 这里放引用内容
> 或者让AI处理的素材
```

**代码**

​```
行内代码：`关键词或代码`

代码块：
​```python
print("hello world")
​```
​```

**表格**

```
| 列1   | 列2   | 列3   |
|-------|-------|-------|
| 内容  | 内容  | 内容  |
```

**分割线**

```
---
```

**链接和图片**

```
[链接文字](网址)
![图片描述](图片地址)
```

就这些，真的不难。对照着练一遍，基本就记住了。

## 用什么软件写

工具很多，按使用场景推荐：

**写笔记、建个人知识库**

- **Obsidian**：免费，数据本地存储，支持双链笔记，重度用户首选，数据完全掌握在自己手里
- **Bear**：颜值极高，苹果生态体验丝滑，适合追求简洁的人（订阅制）

**写文章、长内容创作**

- **Typora**：所见即所得，打完符号直接渲染成效果，新手最友好（买断制，约90元）
- **VS Code**：免费，插件生态强大，写文章和写代码都能用

**团队协作**

- **Notion**：支持Markdown输入，协作和数据库功能强，适合团队共享文档
- **飞书 / 语雀**：国内团队主流选择，Markdown支持完善，上手成本低

**不想装软件，直接在线用**

- **StackEdit**：浏览器打开即用，左边写右边实时看渲染效果
- **Markdown Nice**：专为公众号排版设计，写完一键复制，格式完美保留，内容创作者必备

**AI工具本身就是最好的练习场**：直接在ChatGPT、Claude里用Markdown格式写提示词，感受立竿见影的回答质量变化。

## 去哪里学

从零开始，这几个资源足够：

**先过一遍基础语法**

- **Markdown Tutorial**（markdowntutorial.com）：交互式教程，边学边练，半小时搞定基础
- **CommonMark官方教程**（commonmark.org）：权威规范出品，讲解准确清晰

**当手册随时查**

- **Markdown Guide**（markdownguide.org）：语法速查表，写作中途忘了什么直接来翻
- **GitHub官方文档**：专讲GitHub风格Markdown（GFM），在GitHub上写文档必看

**延伸了解背景**

- John Gruber的博客（daringfireball.net）：Markdown创始人本人写的原版设计思路
- 纪录片《互联网之子》：Aaron Swartz的故事，看完对Markdown会有完全不一样的感受

## 最后说一句

Markdown不复杂，难的是开始用的那一步。

很多人觉得"我又不是程序员，学这个干嘛"——但现在的情况是，你每天用的AI本身就在用Markdown输出内容，你不懂它，就像收到一封信却看不懂标点符号在哪断句。

从今天起，选一个工具，写一篇东西——周报、读书笔记、随手想法都行。

用过的人都说同一句话：就这？这么简单？

对，就这么简单。
